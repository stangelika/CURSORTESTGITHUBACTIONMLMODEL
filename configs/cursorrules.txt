# Правила проекта для Cursor

Язык общения: русский.

## Общие принципы
- Делаем минимальные, безопасные и проверяемые изменения. Если требования неясны — уточняй перед правками.
- Соблюдай текущий стиль кода и форматирование (lint/formatter), не отключай правила без причины.
- Не включай в контекст и не изменяй файлы, перечисленные в `.cursorignore`.
- Никогда не коммить секреты, ключи, токены или приватные данные.
- Изменения сопровождай понятными коммитами в стиле Conventional Commits (см. правила в `.cursor/rules/commit-style.md`).
- Предпочитай локальные, узконаправленные правки; большие изменения дели на логические шаги.

## Область изменений
- Если не указано иное, изменяй код только в директориях `src/`, `app/`, `lib/` (если существуют). Конфиги и инфраструктуру правь отдельно и осознанно.
- Не изменяй автоматически сгенерированные файлы и артефакты сборки.

## Тестирование и качество
- При добавлении функционала добавляй/обновляй тесты.
- Не ломай существующие тесты без веской причины и объяснения.

## Что не делать
- Не меняй публичные API без согласования.
- Не внедряй новые крупные зависимости без обсуждения.

## MCP Серверы
### Context7 MCP
- **Всегда используй Context7** для получения актуальной документации при работе с кодом
- Автоматически применяй инструменты `resolve-library-id` и `get-library-docs` без явного запроса
- При любых вопросах по библиотекам, API, примерам кода или конфигурации проектов - обращайся к Context7
- Это обеспечивает актуальную информацию вместо устаревших данных из обучающих материалов

### GitHub MCP
- **Всегда используй GitHub MCP** для работы с репозиториями, issues, pull requests и GitHub Actions
- Автоматически применяй GitHub MCP инструменты при вопросах о:
  - Анализе кода в репозитории
  - Управлении issues и PR
  - Мониторинге CI/CD процессов
  - Проверке безопасности кода
  - Командной работе и обсуждениях
- При работе с Git/GitHub всегда используй MCP вместо обычных команд git
